# Настройка парсера nsbonline.ru

## Проблема
Парсер запускается, но не находит данные (`No data parsed. Check selectors.`).
Это потому, что CSS селекторы не настроены под реальную структуру сайта.

## Решение

### Шаг 1: Откройте nsbonline.ru

Перейдите на https://nsbonline.ru в браузере.

### Шаг 2: Найдите CSS селекторы

1. Нажмите правой кнопкой на элементе с домашним заданием
2. Выберите "Inspect" / "Просмотреть код"
3. Найдите следующие элементы и скопируйте их CSS селекторы:

| Элемент | Описание | Пример селектора |
|---------|----------|------------------|
| `.homework-container` | Контейнер всех ДЗ | `.content`, `#homework` |
| `.homework-item` | Одно ДЗ | `.post`, `.item`, `.entry` |
| `.date` | Дата | `.date`, `.time`, `time` |
| `.file-link` | Ссылка на файл | `.attachment`, `a[href*=".jpg"]` |
| `.content` | Текст ДЗ | `.text`, `.description` |
| `.task` | Одно задание | `li`, `.task`, `.question` |

### Шаг 3: Обновите parser-config.ts

Откройте `scripts/parser-config.ts` и обновите селекторы:

```typescript
export const parserConfig = {
  selectors: {
    homeworkContainer: '.YOUR_SELECTOR_HERE',
    homeworkItem: '.YOUR_SELECTOR_HERE',
    date: '.YOUR_SELECTOR_HERE',
    // и т.д.
  },
};
```

### Шаг 4: Запустите парсер

```bash
npm run parse
```

### Шаг 5: Проверьте результат

Откройте `src/app/data/homeworkData.ts` и проверьте, что данные обновились.

---

## Альтернатива: Ручное добавление

Если парсинг не работает, добавляйте ДЗ вручную через админ-панель:

1. Откройте `/admin`
2. Перейдите в "Домашние задания"
3. Нажмите "Добавить ДЗ"
4. Заполните форму

Или редактируйте `src/app/data/homeworkData.ts` напрямую.

---

## Почему парсер может не работать?

1. **SSL сертификат просрочен** ✅ (уже исправлено)
2. **Селекторы не верны** - нужно обновить
3. **Структура сайта изменилась** - нужно адаптировать парсер
4. **CORS / блокировки** - использовать proxy

---

## Быстрый старт без парсера

Просто используйте админ-панель для добавления ДЗ - это надёжнее!
